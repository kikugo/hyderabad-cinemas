# Security Policy

## Supported Versions
Security fixes are provided on the `master` branch.

## Reporting a Vulnerability
- Email: security@kikugo.com (placeholder)  
- Please include: repro steps, expected vs actual, impact, and any logs/screenshots.
- Do **not** open public issues for security reports.

## Response
- Acknowledge within 72 hours.
- Triage and assess impact.
- Patch and release as soon as possible.

## Best Practices
- Never commit secrets or API keys.
- Use `.env` files locally; `.env` is gitignored.
- Rotate keys if exposed.
